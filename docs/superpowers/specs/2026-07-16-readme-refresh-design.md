# README 实用型更新设计

## 目标

更新仓库根目录的 `README.md`，让第一次访问项目的读者能立即打开在线网站、了解课程内容，并在需要时完成本地运行、测试和部署。

## 更新范围

- 在标题和项目简介之后加入醒目的在线体验入口，链接到 `https://nandeqiyu.github.io/fourier-learning-site/`。
- 增加课程内容概览，准确列出六章主题。
- 保留并整理主要功能、本地运行、测试与构建、音频说明。
- 增加 GitHub Pages 部署说明，说明推送到 `master` 分支后由现有工作流自动构建和发布。
- 仅修改 `README.md`；不改课程代码、构建配置和内部设计文档。

## 内容结构

1. 项目标题与一句话介绍
2. 在线体验
3. 课程内容
4. 主要功能
5. 本地运行
6. 测试与构建
7. 部署
8. 音频说明

## 验收标准

- 在线体验链接指向当前可访问的网站。
- 六章名称与 `src/content/chapters.ts` 一致。
- 命令与 `package.json` 中的脚本一致。
- 部署描述与 `.github/workflows/deploy.yml` 一致。
- Markdown 标题层级、列表和代码块排版正确，内容完整。
